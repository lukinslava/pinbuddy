# Дизайн: веб-приложение (PWA) для публикации в Pinterest с AI-описаниями

**Дата:** 2026-07-14 (разворот с нативного iOS на веб)
**Статус:** Утверждён пользователем (платформа/ключ/хостинг)

## 1. Обзор

Мобильное веб-приложение (PWA на React + TypeScript + Vite), которое открывается
по ссылке в Safari на iPhone. Пользователь выбирает фото и короткие видео из
галереи, приложение генерирует для каждого медиа русскоязычные заголовок,
описание и теги под правила Pinterest через Claude API (вызов напрямую из
браузера), даёт отредактировать текст и передаёт медиа в Pinterest через системный
share (Web Share API), положив описание в буфер обмена.

Ниша аккаунта — **lifestyle**. Язык описаний — **русский**. Один пользователь.

## 2. Цели и не-цели

### Цели
- Выбрать пачку фото/коротких видео из галереи телефона (мобильный Safari).
- Сгенерировать для каждого файла заголовок (≤100 симв.), описание с ключевыми
  словами (≤500 симв.) и 3–8 тегов через Claude API.
- Дать отредактировать текст перед публикацией.
- Передать медиа в Pinterest через `navigator.share`, положив текст в буфер.
- Устанавливаться «как приложение» через «На экран „Домой“».

### Не-цели (в этой версии)
- Автопубликация через Pinterest API (отложено, §10).
- Бэкенд/прокси для ключа (ключ хранится в браузере — вариант «direct browser»).
- Аутентификация, мультиаккаунт, планировщик, аналитика.
- Полноценный офлайн-режим (SW — минимальный, только для установки/кэша шелла).

## 3. Ограничения и допущения

- **Claude из браузера.** Anthropic Messages API вызывается напрямую с заголовком
  `anthropic-dangerous-direct-browser-access: true`. Ключ хранится в
  `localStorage` устройства пользователя; расходы идут на аккаунт владельца ключа.
  Модель по умолчанию — `claude-haiku-4-5-20251001` (переключаемо на Sonnet).
- **Публикация.** Через Web Share API (`navigator.share({ files, text })`): в
  Pinterest уходит медиафайл; описание также кладётся в буфер обмена, потому что
  Pinterest, как правило, не заполняет поле описания из share. Пользователь
  вставляет описание одним тапом.
- **Видео через share на iOS.** Работает не всегда стабильно (зависит от размера/
  формата). Предусмотреть запасной путь: кнопка «Скачать видео» + «Открыть
  Pinterest», если `navigator.canShare({ files })` вернул false для видео.
- **Хостинг.** Cloudflare Pages (бесплатно). Статическая сборка Vite. HTTPS
  обязателен (Web Share API и clipboard требуют защищённого контекста).
- **Безопасность ключа.** Ключ в `localStorage` доступен только в браузере
  пользователя; по публичной ссылке чужой видит пустое приложение с запросом
  ключа. Приемлемо для личного использования.

## 4. Архитектура

SPA на React + TypeScript (Vite). Логика вынесена в чистые модули (тестируются
Vitest), браузерные API изолированы в тонких сервисах и React-компонентах.

- **keyStore.ts** — чтение/запись/очистка API-ключа в `localStorage`.
- **settings.ts** — ниша/язык/модель в `localStorage` (с дефолтами).
- **captionRules.ts** — лимиты Pinterest: обрезка title до 100, description до
  500; нормализация тегов (`#`, уникальность, максимум 8). Чистая логика.
- **promptBuilder.ts** — системный и пользовательский промпт из настроек.
- **responseParser.ts** — извлечение JSON `{title,description,tags}` из текста
  ответа (терпит code-fence и лишние пробелы).
- **claudeClient.ts** — `fetch` к `https://api.anthropic.com/v1/messages` с
  нужными заголовками; маппинг ошибок (401/429/прочее); за интерфейсом
  `ClaudeClient` для моков.
- **imageDownscaler.ts** — уменьшение изображения через `<canvas>` (длинная
  сторона ≤1024) → base64 JPEG.
- **frameExtractor.ts** — фото → один кадр; видео → до 3 кадров (0, середина,
  ближе к концу) через `<video>` + canvas.
- **captionGenerator.ts** — оркестрация: кадры → промпт → `ClaudeClient` →
  `captionRules.apply`.
- **shareService.ts** — `captionText()` (чистая склейка) + `share()` (буфер +
  `navigator.share`); фолбэк для видео.
- **Компоненты:** `App` (навигация, гейт на наличие ключа), `SettingsView`,
  `GalleryView` (файл-инпут + список), `CaptionCard` (превью, редактируемые поля,
  счётчики символов, «Сгенерировать»/«Повторить», «Опубликовать»).
- **PWA:** `manifest.webmanifest` (иконки, `display: standalone`) + минимальный
  сервис-воркер для установки.

Каждый модуль имеет одну ответственность и тестируется изолированно; `ClaudeClient`
за интерфейсом, чтобы подменять в тестах и позже заменить на Pinterest API.

## 5. Экраны и поток

1. **Настройки:** ввод API-ключа (маскированное поле) → `localStorage`; ниша
   (default «lifestyle»), язык (default «русский»), модель. Пока ключа нет —
   на галерее баннер со ссылкой сюда.
2. **Галерея:** `<input type="file" accept="image/*,video/*" multiple>` —
   выбор фото/видео; кнопка «Сгенерировать все»; список карточек.
3. **Генерация:** по каждому файлу извлекаются кадры, шлётся в Claude; статус
   (в очереди/генерируется/готово/ошибка).
4. **Правка:** карточка — превью + редактируемые title/description/tags со
   счётчиками символов относительно лимитов.
5. **Публикация:** «Опубликовать» → текст в буфер + `navigator.share` с файлом →
   пользователь выбирает Pinterest и вставляет описание.

## 6. Поток данных

`<input file>` → `File[]` → `frameExtractor` (кадры, canvas) →
`imageDownscaler` (base64 JPEG) → `claudeClient` (Anthropic) → `responseParser`
→ `captionRules.apply` → правка в UI → `shareService` (буфер + share).

## 7. Промпт (правила Pinterest)

Системный промпт кодирует: SEO (ключевые слова в заголовке/описании, релевантные
хэштеги), тех-лимиты (title ≤100, description ≤500, 3–8 тегов), контент-политику
(без спама/запрещёнки/переспама), стиль lifestyle на русском, и требование вернуть
СТРОГО валидный JSON `{"title","description","tags":[...]}`.

## 8. Обработка ошибок

- Нет ключа → баннер и переход в Настройки.
- 401 → «Проверьте ключ»; 429 → «Слишком много запросов, повторите позже».
- Сеть/таймаут → сообщение + «Повторить» на карточке.
- Ответ не парсится в JSON → одна повторная попытка, затем ручная правка.
- `navigator.share` недоступен/видео не шарится → фолбэк «скачать + открыть
  Pinterest» и текст уже в буфере.
- Большие изображения → предварительное сжатие через canvas.

## 9. Тестирование

- **Vitest (чистая логика):** `captionRules` (лимиты, теги), `promptBuilder`
  (наличие правил/лимитов/языка/JSON), `responseParser` (валидный/битый/с
  code-fence), `claudeClient` (мок `fetch`: успех, 401, 429), `captionGenerator`
  (оркестрация со стаб-клиентом), `shareService.captionText` (склейка).
- **Ручная проверка в мобильном Safari:** выбор из галереи, генерация, правка,
  share в Pinterest, буфер, фолбэк для видео, установка на экран «Домой».

## 10. Отложено на будущее

- Автопубликация через Pinterest API v5 (нужен бизнес-аккаунт + одобрение API).
  `shareService` за интерфейсом — можно добавить `PinterestApiService`.
- Мини-прокси на Cloudflare Worker для сокрытия ключа (если приложение станет
  не только личным).
- Планировщик, аналитика, мультиязычные описания, офлайн-режим.

## 11. Открытые вопросы

- Поведение Web Share для видео на конкретной версии iOS — проверить на реале,
  при необходимости включить фолбэк по умолчанию для видео.
- Актуальный id/прайсинг модели Anthropic — держать id в настройках.
