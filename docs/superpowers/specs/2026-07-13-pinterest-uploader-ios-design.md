# Дизайн: iOS-приложение для публикации в Pinterest с AI-описаниями

**Дата:** 2026-07-13
**Статус:** Черновик на утверждение

## 1. Обзор

Нативное iOS-приложение (SwiftUI), которое помогает публиковать фото и короткие
видео из галереи телефона в аккаунт Pinterest, автоматически генерируя для каждого
медиафайла описание и теги через Claude API. Приложение раздаётся одному
пользователю через TestFlight.

Ниша аккаунта — **lifestyle**. Язык описаний — **русский**.

## 2. Цели и не-цели

### Цели
- Выбрать пачку фото и/или коротких видео из галереи.
- Для каждого файла сгенерировать через Claude API: заголовок, описание с
  ключевыми словами, набор хэштегов/тегов — с учётом правил Pinterest.
- Дать отредактировать сгенерированный текст перед публикацией.
- Упростить публикацию в Pinterest настолько, насколько это возможно без
  официального API (share-sheet + буфер обмена).

### Не-цели (в этой версии)
- Автоматическая публикация через Pinterest API (осознанно отложено, см. §10).
- Планировщик/отложенный постинг.
- Мультиаккаунт, роли, публикация для чужих пользователей.
- Публикация в App Store (только TestFlight).

## 3. Ограничения и допущения

- **Pinterest.** Публикация идёт через системный share-sheet: в Pinterest
  передаётся само медиа, а поле описания официальное приложение Pinterest, как
  правило, не позволяет заполнить извне. Поэтому текст копируется в буфер обмена,
  и пользователь вставляет его вручную одним тапом.
- **Claude API.** Ключ Anthropic вводится один раз в настройках и хранится в
  Keychain телефона (вариант «ключ на устройстве»). Расходы идут на аккаунт
  владельца ключа. Рекомендуемая модель по умолчанию — **Claude Haiku 4.5**
  (поддерживает анализ изображений, дёшево и быстро) с возможностью переключить
  на Sonnet для более «богатых» описаний.
- **Распространение.** TestFlight, что требует платного Apple Developer аккаунта
  ($99/год). Сборка в Xcode на macOS.
- **Видео.** Для анализа видео извлекается несколько кадров (обложка + 1–2 кадра
  из середины), а не весь ролик — дешевле и достаточно для сути.

## 4. Архитектура

Чистое SwiftUI-приложение, минимум внешних зависимостей. Основные модули:

- **KeychainStore** — безопасное хранение API-ключа Claude. Единственная
  ответственность: чтение/запись/удаление ключа. Интерфейс: `save(key)`,
  `readKey() -> String?`, `deleteKey()`.
- **SettingsStore** — не-секретные настройки (ниша, язык, выбранная модель) в
  UserDefaults.
- **MediaItem** — модель одного выбранного медиа: ссылка на ассет, тип
  (фото/видео), извлечённые кадры для анализа, статус генерации, сгенерированные
  и отредактированные поля (title, description, tags).
- **FrameExtractor** — из фото делает подготовленное изображение; из видео
  извлекает 1–3 репрезентативных кадра (`AVAssetImageGenerator`). Ужимает кадры
  до разумного размера перед отправкой в API.
- **ClaudeClient** — обёртка над Anthropic Messages API. Единственная
  ответственность: принять кадры + параметры (ниша/язык/лимиты), вернуть
  структурированный результат (title/description/tags). Знает про формат запроса,
  base64-изображения, обработку ошибок и лимитов.
- **CaptionGenerator** — оркестрация: берёт `MediaItem`, зовёт `FrameExtractor` и
  `ClaudeClient`, парсит ответ в поля модели, обновляет статус.
- **PinterestShareService** — кладёт итоговый текст в буфер обмена и открывает
  share-sheet (`UIActivityViewController`) с медиа для передачи в Pinterest.
- **UI (View-слой)** — экраны из §5, связанные через простые `@Observable`
  view-модели.

Каждый модуль тестируется изолированно; `ClaudeClient` и `PinterestShareService`
прячутся за протоколами, чтобы подменять их в тестах.

## 5. Экраны и поток

1. **Настройки (первый запуск / доступны всегда):**
   - Поле ввода API-ключа Claude → сохраняется в Keychain.
   - Ниша (по умолчанию «lifestyle»), язык (по умолчанию «русский»).
   - Выбор модели (Haiku 4.5 по умолчанию / Sonnet).
2. **Выбор медиа:** системный `PhotosPicker` с множественным выбором фото и
   коротких видео.
3. **Генерация:** для каждого выбранного файла извлекаются кадры и отправляются
   в Claude; показывается статус (в очереди / генерируется / готово / ошибка).
4. **Просмотр и правка:** список карточек — превью + редактируемые поля
   (заголовок, описание, теги). Пользователь может поправить любой текст.
5. **Публикация:** на карточке кнопка «Опубликовать» → текст в буфер, открывается
   share-sheet Pinterest с этим медиа → пользователь вставляет описание и
   публикует в приложении Pinterest.

## 6. Поток данных

`PhotosPicker` → `MediaItem[]` → `FrameExtractor` (кадры) → `ClaudeClient`
(base64-кадры + системный промпт с правилами) → распарсенные поля в `MediaItem`
→ правка в UI → `PinterestShareService` (буфер + share-sheet).

## 7. Промпт для Claude (правила Pinterest)

Системный промпт инкапсулирует все требования и просит вернуть строгий JSON:

- **SEO:** цепляющий заголовок с ключевым словом; описание с 2–3 релевантными
  ключевыми словами естественным языком; релевантные хэштеги.
- **Тех-лимиты:** заголовок ≤ 100 символов; описание ≤ 500 символов; 3–8
  хэштегов (без переспама).
- **Контент-политика:** без спама, без запрещённых/чувствительных тем, без
  вводящих в заблуждение заявлений, без накрутки хэштегами.
- **Стиль/ниша:** тон lifestyle, тёплый и вдохновляющий, на русском языке.
- **Формат ответа:** JSON `{ "title": ..., "description": ..., "tags": [...] }`
  для надёжного парсинга; при неполном ответе — повтор/деградация в понятную
  ошибку.

## 8. Обработка ошибок

- Нет ключа API → баннер и переход в Настройки.
- Ошибка сети / таймаут → сообщение + кнопка «Повторить» на карточке.
- Ошибка/лимит API (429, 401) → понятное сообщение (проверить ключ / подождать).
- Не удалось извлечь кадр из видео → пометить файл, дать пропустить.
- Ответ не распарсился в JSON → одна повторная попытка, затем ручная правка.
- Слишком большой кадр → предварительное сжатие перед отправкой.

## 9. Тестирование

- **KeychainStore:** сохранение/чтение/удаление ключа (на симуляторе/устройстве).
- **FrameExtractor:** извлечение кадров из тестового видео; сжатие изображений.
- **ClaudeClient:** парсинг корректного и «битого» JSON-ответа; маппинг кодов
  ошибок (мокнутая сеть).
- **CaptionGenerator:** оркестрация со стаб-клиентом (успех, ошибка, повтор).
- **Соблюдение лимитов:** обрезка/валидация длины заголовка и описания.
- Ручная проверка сквозного потока на устройстве до раздачи в TestFlight.

## 10. Отложено на будущее (не в этой версии)

- **Автопубликация через Pinterest API v5:** требует бизнес-аккаунт Pinterest и
  одобренную заявку на доступ к API. Позволит заливать пачку пинов без ручных
  действий. Архитектура (`PinterestShareService` за протоколом) оставляет место,
  чтобы позже добавить `PinterestApiService` как альтернативную реализацию.
- Планировщик публикаций, аналитика, мультиязычные описания.

## 11. Открытые вопросы

- Точная модель по умолчанию и приблизительная стоимость за пин (уточнить по
  актуальному прайсингу Anthropic).
- Проверить на реальном устройстве, принимает ли share-extension Pinterest
  видео тем же путём, что и фото; при отличиях — развести обработку.
